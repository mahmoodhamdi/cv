What I use

A snapshot of the hardware, software, and small habits I actually use day-to-day. Updated occasionally — last touched May 2026.

Editor & terminal

  • VS CodePrimary editor. Workspace per project, minimal extensions: ESLint, Prettier, GitLens, Tailwind IntelliSense, Flutter, Error Lens.
  • JetBrains MonoMonospace font with ligatures. JetBrains Nerd Font variant for terminal glyphs.
  • Zsh + Oh My ZshWith git, z, fast-syntax-highlighting, and a custom prompt that shows git branch + Node version.
  • tmuxOne session per project. Three panes: editor / dev server / logs.
  • fzf + ripgrepFuzzy file search and grep — wired into Ctrl-T / Ctrl-R bindings.
  • zoxideSmarter cd. z proj instead of long paths.

Hardware

  • Lenovo Legion 5Daily driver. 16GB RAM, NVMe SSD, dual-boot but mostly stays on Ubuntu.
  • Ubuntu 24.04 LTSPrimary OS. Faster for Docker workflows and node-gyp builds than my Windows partition.
  • External 27" 1440p monitorVertical for code review, horizontal for designs and dashboards.
  • Mechanical keyboard, 75% layoutTactile switches. Function row matters for IDE shortcuts.
  • Logitech MX MasterHorizontal scroll for wide log panels & spreadsheets is non-negotiable.
  • Wired Ethernet when possibleCuts a noticeable amount of "wait, what?" out of WhatsApp calls with clients.

Backend dev

  • Node.js + TypeScriptDefault for most APIs. Strict tsconfig, no any, no implicit return.
  • Express / NestJSExpress for small services, NestJS when DI & modules pay off.
  • MongoDB + MongooseDefault for flexible schemas. PostgreSQL when relations dominate.
  • RedisSessions, rate limits, pubsub fan-out for WebSockets.
  • PinoStructured JSON logging. pino-pretty for local, raw JSON for prod.
  • ZodRuntime validation that also gives me TypeScript types. Worth every byte.

Frontend & mobile

  • Next.js + TailwindApp Router. Tailwind for components, occasional CSS modules for one-off layouts.
  • Flutter + DartClean Architecture, Bloc for state. Riverpod only when the project starts with it.
  • FigmaFor reading client mockups. I don't design from scratch — that's not my job.

DevOps & deployment

  • Docker + Docker ComposeEvery project ships with a docker-compose.yml that brings up the whole stack.
  • DigitalOcean DropletsCheap, predictable, Linux. SSH + Caddy + Docker = production.
  • VercelFor static + Next.js frontends with no backend logic.
  • CaddyReverse proxy with automatic HTTPS. Simpler config than Nginx for small fleets.
  • GitHub ActionsBuild, lint, test on PR. Deploy to DO via SSH on push to main.
  • CloudflareDNS + CDN + DDoS shield. Free tier covers most of what clients need.

Daily tools

  • Postman / BrunoPostman for collections shared with clients. Bruno for solo work — local files, git-friendly.
  • TablePlusGUI for Mongo + Postgres. Saved queries beat the CLI for one-off investigations.
  • NotionProject notes, client docs, post-mortems.
  • ObsidianPersonal knowledge base + draft blog posts in Markdown.
  • Linear / GitHub IssuesLinear for paid project clients. GitHub Issues for OSS contributions.
  • WhatsApp WebLike it or not, the primary client comms channel in the MENA region.

Open source I depend on

  • nanoidEvery ID I generate. Top contributor and grateful daily user.
  • Helmet.jsFirst middleware I import into every Express app. Contributing back keeps me sharp on security defaults.
  • express-validatorInput validation that won't surprise you in production.
  • Mongoose / Pino / ZodThe Node trio. Honestly couldn't do my job without them.
  • adhan_dartPrayer-times for Islamic apps. I'm the top contributor — it's the one I maintain most actively.

If you want to copy my setup verbatim, don't. Pick the parts that solve a problem you actually have today; replace everything else with what you already know. Tooling is the wrong place to spend your novelty budget.

الأدوات اللي بستخدمها

لقطة سريعة من الهاردوير والسوفتوير والعادات الصغيرة اللي بستخدمها يومياً. بحدّثها من وقت للتاني — آخر تحديث مايو ٢٠٢٦.

Editor والـ terminal

  • VS Codeالمحرر الأساسي. Workspace لكل مشروع، extensions بحدّ أدنى: ESLint, Prettier, GitLens, Tailwind IntelliSense, Flutter, Error Lens.
  • JetBrains Monoخط mono مع ligatures. الـ Nerd Font variant للـ terminal.
  • Zsh + Oh My Zshمع git، z، fast-syntax-highlighting، و prompt مخصص يعرض branch ونسخة Node.
  • tmuxSession واحد لكل مشروع. ٣ panes: محرر / dev server / logs.
  • fzf + ripgrepبحث ملفات وgrep — مربوطين بـ Ctrl-T / Ctrl-R.
  • zoxideبديل ذكي لـ cd. z proj بدل المسارات الطويلة.

الهاردوير

  • Lenovo Legion 5الجهاز اليومي. 16GB RAM، NVMe SSD، dual-boot بس الوقت كله على Ubuntu.
  • Ubuntu 24.04 LTSالـ OS الأساسي. أسرع لـ Docker و node-gyp من الـ Windows partition.
  • شاشة خارجية ٢٧" 1440pعمودي لمراجعة الكود، أفقي للتصميمات واللوحات.
  • كيبورد ميكانيكي 75%مفاتيح tactile. صف الـ Function ضروري للـ IDE shortcuts.
  • Logitech MX Masterالـ horizontal scroll للـ logs الواسعة والـ spreadsheets مش قابل للتفاوض.
  • Ethernet لما متاحبيوفّر دقيقة "بتقول إيه؟" في مكالمات الواتساب مع العملاء.

الباك إند

  • Node.js + TypeScriptالـ default لمعظم الـ APIs. tsconfig صارم، مفيش any، مفيش implicit return.
  • Express / NestJSExpress للخدمات الصغيرة، NestJS لما الـ DI و modules تستاهل.
  • MongoDB + Mongooseللـ schemas المرنة. PostgreSQL لما العلاقات هي الأساس.
  • RedisSessions و rate limits و pubsub لـ WebSockets.
  • PinoLogging مهيكل بـ JSON. pino-pretty محلياً، raw JSON في الإنتاج.
  • ZodValidation وقت الـ runtime تدّيني TypeScript types كمان. بيستاهل كل بايت.

الفرونت والموبايل

  • Next.js + TailwindApp Router. Tailwind للـ components، CSS modules أحياناً للـ layouts المخصصة.
  • Flutter + DartClean Architecture، Bloc للـ state. Riverpod بس لو المشروع بدأ بيها.
  • Figmaلقراءة تصاميم العملاء. مش بصمم من الصفر — ده مش شغلي.

DevOps والنشر

  • Docker + Docker Composeكل مشروع بيتشحن معاه docker-compose.yml بيشغّل كل الـ stack.
  • DigitalOcean Dropletsرخيص ومتوقع وLinux. SSH + Caddy + Docker = إنتاج.
  • Vercelللفرونت الستاتيك أو Next.js اللي مفيهوش backend logic.
  • CaddyReverse proxy مع HTTPS تلقائي. Config أبسط من Nginx للمشاريع الصغيرة.
  • GitHub ActionsBuild و lint و test على الـ PR. Deploy على DO عبر SSH على push للـ main.
  • CloudflareDNS + CDN + درع DDoS. الـ free tier بيغطي معظم احتياجات العملاء.

الأدوات اليومية

  • Postman / BrunoPostman للـ collections اللي بشاركها مع العملاء. Bruno لشغلي لوحدي — ملفات محلية وgit-friendly.
  • TablePlusGUI لـ Mongo و Postgres. الـ saved queries أحسن من الـ CLI للفحص السريع.
  • Notionملاحظات المشاريع، توثيق العملاء، post-mortems.
  • Obsidianقاعدة معرفة شخصية + مسودات المقالات بـ Markdown.
  • Linear / GitHub IssuesLinear لعملاء المشاريع المدفوعة. GitHub Issues لمساهمات الـ OSS.
  • WhatsApp Webشئت أم أبيت، هي القناة الأساسية للتواصل مع العملاء في المنطقة العربية.

المصادر المفتوحة اللي بعتمد عليها

  • nanoidكل ID بولّده. مساهم رئيسي ومستخدم يومي ممتن.
  • Helmet.jsأول middleware بستوردها في كل Express app. المساهمة بترجّعني لإعدادات الأمان الصح.
  • express-validatorValidation للـ input مش هتفاجئك في الإنتاج.
  • Mongoose / Pino / Zodالـ Node trio. بصراحة مش هقدر أعمل شغلي من غيرهم.
  • adhan_dartأوقات الصلاة للتطبيقات الإسلامية. أكبر مساهم — هي اللي بصيانتها بشكل أكبر.

لو عاوز تنسخ الـ setup بتاعي بالحرف، ما تعملش. اختار الأجزاء اللي بتحل مشكلة فعلية عندك دلوقتي، واستبدل الباقي بأي حاجة فاهمها أصلاً. الـ tooling مش المكان اللي تستثمر فيه الـ novelty budget.
